5. 태그 특징 정리

1. a 태그 (1) 블록 요소를 자식 요소로 가질 수 있다. (2) a 태그를 부모 요소로 가질 수 없다. 2. ol 태그 (1) start 속성 : 시작 index를 설정해줌 (2) reversed : index가 역순으로 붙음 3. li 태그 (1) 종료 태그를 필요로 하지 않음 (2) value 속성 : 현재 항목의 index값을 설정, 이어지는 li의 index에도 영향을 줌 4. table 태그 (1) tfoot 태그는 필수가 아님 (2) 최소한 하나의 tr 태그는 있어야 함 (3) thead, tbody, tfoot 순서로 와야함 5. colgroup 태그 (1) 시작태그와 종료태그 생략 가능 (2) 종료 태그만 생략 할 수 도 있음 6. thead 태그 (1) tbody나 tfoot태그가 뒤따르면 종료 태그 생략 가능 7. tfoot 태그 (1) 종료태그 생략 가능 8. tr,td,th 태그 (1) 종료태그 생략 가능 9. 헤딩태그 (1) 블록 요소를 자식요소로 가질 수 없음 10. em (1) 문장의 의미를 바꿀때 사용 (강세) 11. strong (1) content에서 가장 중요한 부분에 사용 12. meta (1) 빈태그 이므로 자식요소 없어야 함 (2) 종료 태그 없음 13. html, head, body (1) 주석이 뒤따라 오지 않는다면 시작 태그, 종료 태그를 생략할 수 있음 14. title (1) title 을 쓰지 않으면 브라우저가 페이지의 나머지 부분을 무시함 15. p (1) 블록 요소를 자식요소로 가질 수 없다. (2) 부모가 a 태그가 아니고 섹셔닝 요소가 이어지는 경우 종료 태그를 생략할 수 있음

10. 깃 명령어 빈칸채우기

1. git remote add origin <저장소 url> 2. git branch -d <삭제할 브랜치> : 삭제할 브랜치가 현재 브랜치에 합쳐졌을 경우 3. git branch -D <삭제할 브랜치> : 삭제할 브랜치가 현재 브랜치에 합쳐지지 않아도 가능 4. git diff <시작지점> <끝지점> : 두 지점의 차이를 보여줌 5. git merge <브랜치 이름> : 현재 브랜치에 해당 브랜치를 병합 6. git reset --hard <커밋주소> : 돌아가려는 이력이후의 모든 내용을 지워 버림 7. git reset --soft <커밋주소> : 돌아가려는 이력이후의 커밋을 취소, 바로 다시 커밋 가능 8. git reset --mixed <커밋주소> : 돌아가려는 이력이후의 add를 취소, add 가능한 상태 9. git reset <옵션> HEAD~<숫자> : 현재 위치에서 상대적인 위치만큼 reset 가능 10. git revert <커밋주소> : 해당 주소의 커밋을 삭제, revert 기록과 해당 커밋 기록은 log에 남음
